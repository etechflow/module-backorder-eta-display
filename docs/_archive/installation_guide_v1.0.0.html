Backorder Shipping Restrictor — Installation Guide

Version 1.0.1 · for Magento Open Source / Adobe Commerce 2.4.4+

This guide walks you through installing the module, entering your licence key, and configuring which shipping methods to restrict. Estimated time: 10 minutes.

1. Prerequisites

ComponentRequired version
Magento Open Source or Adobe Commerce2.4.4 or newer
PHP8.1, 8.2, 8.3, or 8.4
MySQL / MariaDBPer Magento requirements
Composer (for Option A)2.x
Hyvä Checkout (optional)1.0+ — if you use it, our Hyvä-specific notice auto-activates

2. Installation

Option A — Composer (recommended)

composer require etechflow/module-backorder-shipping-restrictor:^1.0
bin/magento module:enable ETechFlow_BackorderShippingRestrictor
bin/magento setup:upgrade
bin/magento setup:di:compile
bin/magento cache:flush

Option B — Manual install from zip

Step B-1. Unzip etechflow-module-backorder-shipping-restrictor-1.0.1.zip into your Magento install at: 
<magento-root>/app/code/ETechFlow/BackorderShippingRestrictor/
Important — case-sensitive directory name. The folder must be named ETechFlow (capital E, capital T, capital F). On Linux servers, eTechFlow (lowercase e) is a different directory and will not work. On macOS the filesystem is case-insensitive so it appears to work locally, but breaks when deployed.
Step B-2. Enable the module and run setup: 
bin/magento module:enable ETechFlow_BackorderShippingRestrictor
bin/magento setup:upgrade
bin/magento setup:di:compile
bin/magento cache:flush
Step B-3. Verify the module is enabled: 
bin/magento module:status | grep BackorderShippingRestrictor

You should see ETechFlow_BackorderShippingRestrictor listed under "List of enabled modules".

3. Enter your licence key

In the admin panel, navigate to:

Stores → Configuration → eTechFlow → Backorder Shipping Restrictor

Expand the Licence section. Paste the key from your purchase email into the Licence Key field. Click Save Config.

Don't have a licence key yet? The module runs for free on dev/staging environments. If your Magento install is reachable at localhost, *.test, *.local, staging.*, dev.*, *.magento.cloud, ngrok tunnels, or RFC 1918 private IPs, no licence key is needed.

4. Configure the shipping methods to remove

Still on the same admin page, expand the General section:

FieldWhat to enter
EnabledYes
Shipping Method Codes Multi-select. Choose the shipping methods you want REMOVED when a backorder item is in the cart. Format: carrier_method.

Common examples:
  • ups_NextDayAir — UPS Next Day Air
  • fedex_FEDEX_2_DAY — FedEx 2-day
  • royalmail_tracked24 — Royal Mail Tracked 24
  • dpd_NextDay — DPD Next Day
Skip Drop-Ship Products Set to Yes if you also have ETechFlow_NextDayEligibility installed and want drop-shipped products to be exempt from the restriction.

Set to No if you don't have NextDayEligibility, or if you want the restriction to apply regardless.

Click Save Config.

5. Configure the customer notice

Expand the Notice section:

FieldRecommended value
Show NoticeYes
Notice Style warning (amber) — standard, matches "something requires your attention".
info (blue) — neutral informational.
error (red) — strong signal.
Notice Title Example: Express shipping unavailable
Notice Message Example: One or more items in your cart is on backorder. Standard shipping is available — express options have been removed.

Click Save Config.

6. Verify it's working

  1. Find a product currently on backorder (or temporarily mark one Out of Stock + enable backorders)
  2. Add it to a cart
  3. Proceed to checkout
  4. On the shipping step, you should see:
    • The notice at the top with your configured title and message
    • The shipping rate list without the methods you configured in Shipping Method Codes
    • Other shipping methods (standard delivery, etc.) still available
  5. Remove the backorder item from the cart (or replace it with an in-stock product)
  6. Reload checkout — the notice should disappear, and all shipping methods including express should be available

If anything doesn't appear, check var/log/exception.log for entries containing etechflow. Most issues are licence-key related or cache-related (try bin/magento cache:flush).

7. Luma vs Hyvä Checkout

The module supports both checkouts automatically:

  • Standard Luma (Knockout) checkout: notice is injected via a ConfigProvider into window.checkoutConfig. Rendered by a Knockout component.
  • Hyvä Checkout: notice is rendered server-side via a Tailwind .phtml template with Alpine.js for the dismiss button. Dark-mode and reduced-motion variants included.

You don't need to switch modes — the module detects the active checkout and renders accordingly.

8. Common installation issues

SymptomFix
Setup:upgrade fails with "module not found" Directory must be ETechFlow (capital ETF), not eTechFlow. Rename and retry.
setup:di:compile fails on this module Almost always a PHP version issue. Verify php -v shows 8.1+. Magento DI compile uses CLI PHP.
Notice doesn't appear in standard Luma checkout Check window.checkoutConfig.backorderRestrictor in browser console. If undefined when cart has a backorder item: cache flush + setup:static-content:deploy -f. Most common cause: leftover stale frontend assets.
Notice doesn't appear on Hyvä Checkout Verify Hyvä Checkout is the active checkout. Verify the template path exists. Verify Hyvä's tailwind.config.js content array includes our template path (Hyvä's default config does).
Shipping methods aren't being filtered even though notice appears The notice and the filtering use the same plugin, so if the notice shows, the plugin IS running. Check that your Shipping Method Codes match the exact code format Magento uses for those methods (carrier_method with the right case and underscores).
"Skip Drop-Ship Products" toggle has no effect This toggle requires ETechFlow_NextDayEligibility to also be installed (it provides the drop_ship_eligible attribute). Without NextDayEligibility, the module silently treats all products as non-drop-ship.

9. Uninstall

bin/magento module:disable ETechFlow_BackorderShippingRestrictor
composer remove etechflow/module-backorder-shipping-restrictor    # if installed via Composer
# OR
rm -rf app/code/ETechFlow/BackorderShippingRestrictor              # if installed manually
bin/magento setup:upgrade
bin/magento cache:flush
Note: This module does not add any product attributes or database tables, so uninstall is clean — nothing to drop from your database.
